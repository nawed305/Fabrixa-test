====================================================
AUTO-DISCOVERY SYSTEM
====================================================
New models added to src/assets/models/ are automatically discovered at runtime via
garmentAssets.ts using Vite's import.meta.glob. If no matching definition exists in
garments.ts or garmentCatalog.ts, a generic single-part garment is created automatically.

To get full per-part texturing for a new model:
1. Add the .glb file to src/assets/models/
2. Add a GarmentType entry in src/lib/fabrixa/garments.ts (or garmentCatalog.ts)
   with:
   - id: unique identifier (snake_case)
   - label: display name
   - emoji: single emoji
   - gender: "men" | "women" | "unisex"
   - parts: array of { id, label, defaultColor }
   - modelPath: "models/filename.glb"
   - meshMap: node-name-substring → part-id mapping
3. The garment will automatically appear in the dashboard menu under the correct
   gender group, with all part-editing, texture, AI-generation, and lasso-selection
   features available.

====================================================
MESH MAPPING STRATEGY
====================================================
The meshMap uses substring matching (case-insensitive) on GLB node names.
Keys are sorted by length (longest first) for specificity.
Fallback order: node name → parent name → material name → positional index.

Recommended GLB naming convention for new models:
  - body / torso / main
  - sleeve / arm / shoulder
  - collar / neck / neckline
  - cuff / ankle / wrist
  - hem / border / trim / edge
  - waist / waistband / band / belt
  - leg / legs / pant / bottom
  - pocket / pouch
  - button / stud / closure
  - hood / cap
  - lapel / flap / facing
  - blouse / top / bodice
  - skirt / flare / bottom
  - dupatta / drape / scarf
  - placket / panel / front
